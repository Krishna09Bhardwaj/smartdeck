# SmartDeck — AI-Powered Flashcard Engine

> **Cuemath AI Builder Challenge — Round 2**
> Built by [Krishns09Bhardwaj](https://github.com/Krishns09Bhardwaj)

**Live URL:** https://smartdeck-delta.vercel.app

SmartDeck turns any PDF — a textbook chapter, class notes, revision material — into a smart deck of flashcards generated by Gemini AI. The app then uses the **SM-2 spaced repetition algorithm** to schedule reviews per card, so students always know exactly what to study next.

---

## Demo Flow

1. Sign up → upload a PDF → Gemini generates 15–25 flashcards in ~15 seconds
2. Click **Study** → flip each card → rate recall (Forgot / Hard / Good / Easy)
3. SM-2 schedules the next review date per card automatically
4. Dashboard shows due counts; Progress page shows mastery % per deck

## Architecture Decisions

### API key security
All Gemini and Supabase service-role calls happen exclusively inside `app/api/` routes. `lib/claude.ts` and `lib/supabase/server.ts` are never imported by any client component. A grep over the entire `components/` and `app/(app)/` tree for `GEMINI` and `service_role` returns zero results.

### SM-2 as a pure function
`lib/sm2.ts` contains zero side effects — it takes `{ease_factor, interval_days, repetitions, quality}` and returns the next state + review date. This makes it trivially unit-testable (8 tests, all passing) and trivially auditable. The API route (`app/api/reviews/route.ts`) calls it and upserts the result.

### Lazy review rows
`card_reviews` rows are created on the **first review**, not at card creation. This keeps the generate flow fast and avoids inserting N rows of default state that would never be read until the student starts studying.

### Supabase RLS
Every table has row-level security enabled. Policies restrict `SELECT`, `INSERT`, `UPDATE`, and `DELETE` to `auth.uid() = user_id`. Cards are protected indirectly — their policies join through `decks` to verify ownership. No data leaks between users are possible even if the API route has a bug.

### PDF text truncation
Extracted text is capped at 15,000 characters before being sent to Claude. This prevents token overruns on dense chapters while still giving Claude enough material to generate 15–25 high-quality cards. Scanned PDFs (< 100 chars extracted) are rejected with a clear user-facing error.

### Mastery threshold
A card is considered "mastered" when its SM-2 `interval_days >= 21`. This means the algorithm has pushed it out at least three weeks — a defensible proxy for long-term retention.

## Local Setup

### Prerequisites
- Node.js 18+
- A [Supabase](https://supabase.com) project
- A [Gemini](https://aistudio.google.com) API key (free)

### Steps

```bash
# 1. Clone the repo
git clone https://github.com/Krishns09Bhardwaj/smartdeck.git
cd smartdeck

# 2. Install dependencies
npm install

# 3. Configure environment
cp .env.example .env.local
# Fill in your Supabase URL, anon key, service role key, and Gemini API key

# 4. Run the database schema
# Go to Supabase dashboard → SQL Editor → paste contents of supabase/schema.sql → Run

# 5. Start the dev server
npm run dev
# Open http://localhost:3000
```

| Variable | Where to find it |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase dashboard → Settings → API → Project URL |

| `SUPABASE_SERVICE_ROLE_KEY` | Supabase dashboard → Settings → API → service_role |
| `GEMINI_API_KEY` | aistudio.google.com → Get API Key |

## SM-2 Algorithm

After each card review, the student rates recall quality (mapped to 1/3/4/5):

| Button | Quality | Meaning |
|---|---|---|
| Forgot | 1 | Complete blank |
| Hard | 3 | Correct but required heavy effort |
| Good | 4 | Correct with minor hesitation |
| Easy | 5 | Perfect recall |

The algorithm updates three values per card per user:
- **ease_factor** (default 2.5, min 1.3) — difficulty multiplier
- **interval_days** — days until next review (1 → 6 → grows exponentially)
- **repetitions** — consecutive correct reviews

Cards rated < 3 reset to interval=1. Cards rated ≥ 3 grow their interval. The `next_review_date` stored in `card_reviews` drives everything — a card is "due" if that date ≤ today.
